# CookIt - Веб-приложение для управления рецептами

**CookIt** - это полнофункциональный веб-сайт для управления кулинарными рецептами с возможностью регистрации пользователей, аутентификацией, CRUD-операциями, фильтрацией рецептов и системой избранного.

## Цель проекта
Разработка полнофункционального веб-сайта для управления рецептами с:
- Регистрацией пользователей и аутентификацией
- Базовыми CRUD-операциями для рецептов
- Фильтрацией рецептов по различным параметрам
- Системой избранного
- Подтверждением email пользователей

## Выполненные задачи
1. Развернута базовая инфраструктура проекта
2. Реализована система рецептов со связанным функционалом
3. Внедрена безопасная аутентификация
4. Разработан пользовательский интерфейс
5. Добавлены расширенные функции для удобства пользователей

## Технологический стек

### Бэкенд
- **ASP.NET Core Web API** (высокая производительность, асинхронность, встроенная поддержка Swagger)
- **Entity Framework Core** (подход Code First, миграции)
- **ASP.NET Identity + JWT** (аутентификация с механизмом refresh-токенов)
- **PostgreSQL** (надежная СУБД)

### Фронтенд
- **Vite + React** (быстрая сборка, горячая перезагрузка)
- **Tailwind CSS** (утилитарный подход к стилям)
- **React Router** (навигация)

## Функциональные возможности

### Бэкенд (ASP.NET Core Web API)
- **Система пользователей**:
  - Регистрация с уникальным логином и email
  - Аутентификация с JWT
  - Механизм обновления токенов
  - Подтверждение email
  - Ролевой доступ (админ/пользователь)
  
- **Управление рецептами**:
  - CRUD-операции для рецептов
  - Фильтрация и пагинация
  - Работа с изображениями
  - Система избранного
  - Управление справочниками (типы блюд, ингредиенты, оборудование)

- **Архитектура**:
  - Чистая архитектура с разделением на слои:
    - Core (сущности, DTO, интерфейсы)
    - Application (бизнес-логика)
    - Infrastructure (EF Core, репозитории)
    - API (контроллеры, DI)

### Фронтенд (React)
- **Аутентификация**:
  - Регистрация/вход с валидацией
  - Подтверждение email
  - Механизм обновления токенов

- **Работа с рецептами**:
  - Просмотр списка рецептов (карточки)
  - Детальная страница рецепта
  - Создание/редактирование рецептов
  - Фильтрация по параметрам:
    - Время приготовления (общее/активное)
    - Острота (1-5)
    - Сложность (1-5)
    - Тип блюда
    - Ингредиенты
    - Оборудование
  - Управление избранным

- **UI-компоненты**:
  - Интерактивные рейтинги (визуализация сложности/остроты)
  - Модальные окна подтверждения
  - Адаптивные карточки рецептов
  - Формы с валидацией

- **Архитектурные особенности**:
  - Модульная структура
  - Компоненты: RecipeCard, RecipeForm, VisualRating
  - Сервисы API (authService)
  - Контексты для глобального состояния (AuthContext)
  - Защищенные маршруты
    
# Инструкция по локальному запуску проекта

## Требования
- .NET 8.0 SDK
- Node.js 
- Docker & Docker Compose
- PostgreSQL
- bash (или командная строка)

## Порты приложения
- **Frontend**: http://localhost:5173
- **Backend API**: http://localhost:5000 | https://localhost:7000
- **Swagger документация**: https://localhost:7000/swagger
- **MinIO консоль**: http://localhost:9001 (admin/password)
- **PostgreSQL**: localhost:5432

## Запуск проекта

### 1. Запуск инфраструктуры
```bash
# Выполнить в корне проекта
docker-compose up -d
```
### 2. Настройка БД
**PostgreSQL должна быть установлена и запущена**

### Создание базы данных

**Через командную строку (psql):**
```bash
# Подключение к PostgreSQL
psql -U postgres -h localhost -p 5432

# Создание базы данных (внутри psql)
CREATE DATABASE "CookIt";

# Выход из psql
\q
```
Измените пароль от БД в CookIt.Api/appsettings.json на свой
```bash
# Выполнить в папке CookIt.Api
cd CookIt.Api
dotnet ef database update
```
### 3. Запуск бэкенда
```bash
# Остаемся в папке CookIt.Api
dotnet run
```
### 4. Запуск фронтенда
```bash
# Открыть новый терминал и перейти в папку frontend
cd ../frontend/cookit/src
npm install
npm run dev
```
